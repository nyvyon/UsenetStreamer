UsenetStreamer Control Panel

Configure addon credentials, NNTP triage settings, and grab the manifest URL without editing environment files by hand.

Authentication

Enter your admin token (ADDON_SHARED_SECRET) to access the dashboard. This is the value you set in your Docker/environment config — it cannot be changed here.

Treat this token like a password. If someone has both the token and the public addon URL, they can open the admin panel and view or modify your credentials.

Streaming Mode

Choose how streams are delivered to Stremio. Desktop Native mode requires Stremio v5 on Windows desktop.

⚡ Desktop Native Mode

Streams NZBs directly using your NNTP provider credentials. No NZBDav required. Only works with Stremio v5 on Windows desktop.

Prowlarr/NZBHydra are disabled in this mode — use direct Newznab indexers only.

Provide your Easynews credentials to let the addon search Easynews directly (no external bridge required). Only the username and password are needed.

Desktop Native mode: Easynews results only play if your addon is served over HTTPS (Stremio refuses plain HTTP URLs).

NZBDav Connection

Stream Protection

Protection Modes

Mode How it works Speed Trade-offs
No Protection Plays the stream you click directly. If the NZB has missing articles, NZBDav shows a failure video. ⚡ Instant No safety net — broken NZBs show a failure screen. Stream links stay valid as long as the file exists in NZBDav.
Auto-Advance Only If the stream you click fails in NZBDav, the addon automatically advances to and plays the next available NZB. No health badges are shown. ⚡ Instant start, slow failover You may not always play what you clicked. Auto-advance can take 5–30s while NZBDav processes the next NZB, depending on file size. Not suitable for large files. Sessions expire after 72 hours or when you save config changes.
Upfront Health Check Spends 5–10 seconds verifying NZBs before returning results. Streams show ✅ (verified), ⚠️ (unverified), or 🚫 (blocked) badges — you play exactly what you see. 🕐 5–10s before results, depending on your indexer speeds Slower initial load. You always know what's healthy before clicking. Stream links stay valid as long as the file exists in NZBDav.
Upfront Health Check + Auto-Advance Same upfront health check with badges. In the rare case a verified NZB still fails in NZBDav, the addon automatically advances to the next healthy stream. 🕐 5–10s before results, depending on your indexer speeds Best of both — badges for visibility, auto-advance for safety. Slightly slower initial load.
BG Health Check + Smart Play Health checks run in the background while results load instantly. The "🎯 Smart Play" stream at the top picks the first verified NZB. Normal streams play exactly what you click — no auto-advance, no surprises. If a normal stream fails, you see the failure screen. Badges populate instantly on subsequent requests from cache. Stream links stay valid as long as the file exists in NZBDav. ⚡ Instant results, fastest playback start time You always play what you click. Use this if you are picky about what you play. Only Smart Play itself has built-in auto-advance to the next verified NZB.
BG Health Check + Smart Play + Auto-Advance Same as above, plus auto-advance on all streams. If any stream you click fails in NZBDav, the addon automatically advances to the next available NZB. Smart Play also has its own auto-advance. Badges populate instantly on subsequent requests from cache. ⚡ Instant results, fastest playback start time Maximum protection — you almost never see a failure screen. Trade-off: you may not always play what you clicked when a normal stream fails. Sessions expire after 72 hours or when you save config changes.

Pre-cache Option

Setting Behavior
ON With Smart Play, the fastest-verified NZB is queued to NZBDav immediately. Without Smart Play, the top result is pre-downloaded before you click. Speeds up playback for the top result but may delay other results if NZBDav is busy processing the pre-cached file.
OFF Nothing is queued until you click. Your click gets NZBDav's full attention.

NNTP Health Check Credentials

Health check: Lightweight verification that the required articles are available.

Archive check: Blocks NZBs that look complete but only contain samples, nested archives, or no playable video.

Stream Status Legend

  • ✅ Green: verified — direct video or stored archive (copy-only). Expected to play.
  • ⚠️ Unverified: articles present but archive not inspected (e.g., 7z). Might or might not play.
  • 🚫 Red: blocked — missing articles or unsupported/compressed archive. Will not play.

Mark at least one indexer as paid (manager list or direct Newznab) before running health checks.

TMDb Metadata & Localization (Optional)

When enabled, fetches metadata directly from TMDb (skips Cinemeta) and adds localized title searches for non-English content. Example: A Telugu movie will search with its original Telugu title, English title, and any additional languages you select below.

Additional Languages (Not recommended) Select additional languages to search with, along with auto-detected language. Leave empty to use only auto-detected language.

TVDB ID Mapping (Optional)

Enable to resolve TVDB ↔ IMDb IDs for series. This improves ID-based searches when only one of the IDs is available.
